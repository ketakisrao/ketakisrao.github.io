Problem Identification

User feedback

I was communicated that the users found it difficult to navigate around and significant features were masked behind buttons and accordions which made them difficult to discover without a prior demo. A detailed survey led to the following points:

  1. The map is congested with pins that don't make much sense. There is no way to determine which pin represents corresponding location.
  2. Congested UI; too many operations filled in one place.
  3. The map size is small, therefore it is difficult to plot devices.
  4. Many features are abstracted and are not easy to carry out without prior knowledge.

UI Evaluation

 
Problems
  1. Map pins are crowded and aren't indicative of the type of inventory they contain.
  2. Duplicate actions such as "Search Inventory" are ambiguous and confusing.
  3. Full name of the inventory is visible on hover in a a tooltip.
  4. Add inventory operations are revealed on clicking this button which are inturn buttons to reveal forms. This nesting is confusing and users lose track of what they want to do.
  5. Long list of inventory is visible as you scroll and not at a glance. The padding for each inventory makes the list even longer.
  6. Significant inventory operations like edit & delete and inventory details are masked behind these which is again confusing since the elements do not look clickable.

Solution

Problem correction

 
Improvements
  1. Map pins have more meaningful icons. Also map is zoomed correctly so the pins do not look overcrowded
  2. Duplicate search box has been removed and a suitable graphic is added to prevent ambiguity.
  3. Inventory names are fully visible.
  4. Add inventory changed to dropdown to prevent masking of features.
  5. Long list of Inventory is paginated for a cleaner look. Extra padding of elements is eliminated.
  6. Edit and delete operations are visible and don't require added clicks.

Restructuring Inventory Hierarchy

After understanding the user's requirements, we finalized the following types of inventory:

  1. Location: A point on a google map denoting the physical co-ordinates of the inventory.
  2. Site: An image of a site layout containing multiple devices.
  3. Device: A physical device that logs data which is required to compute our metrics.
  4. Zone: A cut out from a site, which is a special area of interest for data & metric visualization.

Adding Locations

Improvements
  1. Added location categories so as to distinguish between different types of inventory.
  2. Map area increased at the time of addition so as to have a wider area for plotting.
  3. Enabled searching on google map to speed up the add location process. A pin is dropped at the address searched for.
  4. Pins made draggable to adjust location easily & precisely.
  5. Appropriate notification of success/failure post addition.

Adding Sites

Improvements
  1. Widened site map area to precisely edit site image.
  2. Added image editing tools so that site images can be edited on the go, one need not search for editors to crop and rotate images.
  3. Appropriate notification of success/failure post addition.

Adding Devices

Improvements
  1. Enabled zooming(bottom right corner) into images for precise plotting of device pins.
  2. Allows you to add multiple devices in one go, thus saving time.
  3. Implemented draggable pins to precisely adjust the location of the pins.

Adding Zones

Improvements
  1. Zoom and Crop options to easily cut out zones.
  2. Zones need a minimum of 3 devices. Thus, added appropriate error messages on selection of less than 3 devices
